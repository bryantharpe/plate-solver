TL;DR

First focus, lowest risk: C2 / H2 — make SolveFromImage honor the request's detection fields (sigma, binning, noise, …) instead of hardcoding sigma=4.0, noise_estimate=1.0, binning=1. It's a correctness gap (CODEBASE-REVIEW C2 / Phase H H2, the SP4 run log's named "next" task) that also collapses the one real solve-latency outlier: hale_bopp.jpg goes 0.168 s NO_MATCH → sub-ms MATCH_FOUND.

What the change does

Library: ps-solve

  • New DetectParams struct (sigma, noise_estimate, binning, normalize_rows, detect_hot_pixels, return_binned, use_binned_for_star_candidates).
  • Default = today's hardcoded values (4.0 / 1.0 / 1 / false / true / false / false).
  • New solve_from_image_with_detect(db, image, params, detect).
  • Existing solve_from_image → one-line wrapper using DetectParams::default().

→ ps-solve/src/lib.rs

gRPC: ps-grpc

  • SolveFromImage estimates noise from the image (same helper as ExtractCentroids).
  • Threads extract.sigma / binning / detect_hot_pixels / normalize_rows / … into DetectParams.
  • Lifts the effective-binning rule into a shared helper used by both handlers (no drift).

→ ps-grpc/src/service.rs · proto unchanged · ps-detect/src/* untouched

Spec scenarios (the contract)

plate-solver (MODIFIED)

  • Defaults preserve prior behavior — WHEN solve_from_image called without explicit DetectParams THEN detection runs 4.0/1.0/1/false/true/false, byte-identical to before.
  • Explicit params honored — WHEN caller passes sigma:8.0, binning:2 THEN get_stars_from_image receives those.
  • Solve math untouched — WHEN called with default THEN combos_examined identical to pre-change.

grpc-service (MODIFIED)

  • Client sigma honored — extract.sigma=8.0 → detection uses 8.0.
  • Noise estimated, not hardcoded — derived from image, matches ExtractCentroids.
  • Consistency with composed path — same params → same attitude/matched IDs as ExtractCentroids+SolveFromCentroids.
  • Effective binning shared — same rule as ExtractCentroids.
  • No accuracy regression — reference image at sigma=4 still MATCH_FOUND, RA/Dec within 10″.
