## Purpose

**What:** Produce a complete, review-ready OpenSpec documentation set that specifies a from-scratch **Rust** reimplementation of the tetra3/cedar "lost-in-space" plate-solving system — star detection → pattern-database lookup → attitude (RA/Dec/Roll/FOV/distortion) recovery — exposed to consumers over a **gRPC** `PlateSolver` service and embeddable on **mobile** (iOS/Android). The documentation is the deliverable; no Rust code is written in this loop.

**Why:** The reference implementations are Python (tetra3, cedar-solve) plus a Rust star-detector (cedar-detect). A mobile, low-latency product needs the whole pipeline in Rust. Before building, we need an unambiguous, validated spec set (a PRD, one feature per subsystem, and testable specs) so implementation can proceed against a clear contract. The canonical source of truth is `reference-solutions/docs/01..08` (rebuild-level docs) — every feature spec is derived from and cites these.

**Definition of done (observable):** `openspec/` contains `project.md`, `PRD.md`, `STATUS.md`, and **7 active OpenSpec changes** (`01-foundation-math-core`, `02-star-detection`, `03-pattern-database`, `04-database-generation`, `05-plate-solver`, `06-grpc-service`, `07-mobile-runtime`), and **every** change passes `openspec validate <change> --strict` (exit 0) with `openspec status --change <change>` showing **4/4 artifacts** (proposal, specs, design, tasks). Concretely, the loop halts when **all task checkboxes below are `[x]`** and the Phase E validation task confirms a clean `--strict` sweep over all 7 changes.
