## Purpose

**Why:** The reference implementations are Python (tetra3, cedar-solve) plus a Rust star-detector (cedar-detect). A mobile, low-latency product needs the whole pipeline in Rust. Before building, we need an unambiguous, validated spec set (a PRD, one feature per subsystem, and testable specs) so implementation can proceed against a clear contract. The canonical source of truth is `reference-solutions/docs/01..08` (rebuild-level docs) тАФ every feature spec is derived from and cites these.
